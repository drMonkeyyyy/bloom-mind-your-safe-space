# ARCHITECTURE.md

Build clean, modular, maintainable software.

Use reusable components.

Avoid duplicate logic.

Separate:

- UI components
- Business logic
- API services
- Hooks
- Utilities
- Types
- Constants
- Validation schemas

Never create massive files.

Never duplicate components.

Prefer simple architecture over clever architecture.

Every feature must be easy to maintain, test, and extend.
